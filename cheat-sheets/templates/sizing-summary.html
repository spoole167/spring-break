Harder Than It Looks

These changes are sized Small or Medium — but carry hidden complexity that can blow up the estimate.

High Risk — Effort Likely Understated

Card Size Risk Why it's worse than it looks
AspectJ @Observed S HIGH Completely silent. Observability via @Observed stops working. No errors, no warnings. You find out when dashboards go blank in production.
OTLP HTTP Primary S HIGH Silent production failure. Traces/metrics vanish. Requires network policy changes (port 4317→4318) and possibly different vendor endpoints.
Batch Schema Rename S HIGH Not just a rename. Column types and table structures changed. Rolling deploys break. Tests pass on H2 while production fails on real databases.
spring-jcl Removed S HIGH Enterprise POM cascade. The commons-logging exclusion pattern used for a decade is now harmful. Corporate parent POMs break downstream.
Hibernate Processor S HIGH Unbounded compile failures. New processor validates HQL at build time. A one-line GAV change surfaces every broken query in the codebase.
Jackson Locale Format S HIGH Silent data corruption. Locale strings flip from zh_CN to zh-CN. DB equality checks silently fail in i18n-heavy apps.
Gradle Version S MOD Multi-step upgrade. Can't jump straight to 8.14. Staged through major boundaries; Gradle 9 enables config cache by default.
Path Matching Engine M HIGH Security hole. URL patterns that silently stop matching can expose previously-protected endpoints. Not just a broken feature — a vulnerability.
Auth Default Deny M HIGH Infra breakage. Actuator endpoints silently blocked. Health checks, readiness probes, and load balancer monitoring fail in production.
Retryable Txn Order M HIGH Near-impossible to test. AOP ordering change causes rolled-back data before retry. Subtle data consistency issues that unit tests won't catch.
Jackson Dates/Timestamps M MOD Compounding changes. Interacts with the separate date-format change. Two Jackson date shifts at once makes debugging doubly confusing.
Test Slice Relocated S MOD Silent test degradation. Tests may load full application context instead of a thin slice. Tests pass but are slower and less precise.
Pattern: The most dangerous changes are Tier 3 "Wrong Results" and Tier 2 silent failures. They don't break your build — they break your production data, your observability, or your security posture. Budget extra time for these even when the per-card effort is small.
