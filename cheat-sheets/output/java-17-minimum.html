Tier 1 · Won't Build spring-boot 3.5 → 4.0

Java 17 Minimum Requirement

Spring Boot 4.0 requires Java 17 as the minimum language level, with 21 recommended. Projects still on Java 11 or earlier won't compile at all.
Effort: S

What You'll See

$ mvn compile
[ERROR] Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.13.0:compile
  on project my-service: Fatal error compiling: error: release version 17 not supported
[ERROR]
[ERROR] -> [Help 1] Check that your JAVA_HOME points to a JDK 17+ installation.

What Changed

Spring Boot 4.0 sets java.version to 17 as the baseline. The framework bytecode and all starters are compiled at the Java 17 language level. Java 11 and earlier are no longer supported.

// pom.xml — Java version property
<java.version>11</java.version>
<java.version>17</java.version>

// build.gradle — toolchain
java {
    sourceCompatibility = JavaVersion.VERSION_11
}

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(17)
    }
}


// Dockerfile
FROM eclipse-temurin:11-jre
FROM eclipse-temurin:17-jre

Why This Changed

Java 17 is the oldest LTS release still receiving public updates. Targeting it lets the framework use sealed classes, records, pattern matching, and text blocks throughout the codebase.

How to Fix

A
Upgrade your JDK. Install JDK 17 or (recommended) JDK 21 and set JAVA_HOME accordingly. Update java.version in your POM or toolchain block in Gradle.
B
Update CI and container base images. Change your CI pipeline and Dockerfiles to use a 17+ base image. eclipse-temurin:21-jre is a good default.
✓ Verify: java -version shows 17+ and mvn compile succeeds
Scope check: Check java.version in your POM, sourceCompatibility in build.gradle, JAVA_HOME in CI scripts, and base images in Dockerfiles. Any reference to Java 8, 11, or earlier needs updating.

Watch Out

Java 17 is the minimum, but 21 is recommended. If you're going to upgrade anyway, go straight to 21 to avoid a second migration. Virtual threads and the latest GC improvements are worth the jump.

Third-party libraries compiled against Java 8 or 11 will still work on a Java 17+ runtime. The issue is your own source code and build toolchain, not your transitive dependencies.

Spring Boot 4.0 Migration Guide
Spring Boot 4.0 requires Java 17 as the minimum (Spring Framework 7.0 baseline). Java 21 is recommended. The framework uses sealed classes, records, and pattern matching internally. Java 11 support was dropped in Spring Boot 3.0; this change raises the floor from 17 (Boot 3.x) to 17 as a hard minimum with 21 recommended.
