Tier 1 · Won't Build spring-boot 3.5 → 4.0

@PropertyMapping Relocated to test.context

@PropertyMapping moved from org.springframework.boot.test.autoconfigure.properties to org.springframework.boot.test.context. Old import doesn't compile.
Effort: S

What You'll See

$ mvn test-compile
[ERROR] /src/test/java/com/example/CustomTestAnnotation.java:[3,62]
  error: package org.springframework.boot.test.autoconfigure.properties does not exist

What Changed

@PropertyMapping moved from org.springframework.boot.test.autoconfigure.properties to org.springframework.boot.test.context. Behaviour is unchanged.

// Import on custom test annotation
import org.springframework.boot.test.autoconfigure.properties.PropertyMapping;
import org.springframework.boot.test.context.PropertyMapping;

Why This Changed

Spring Boot 4.0 consolidated test-context infrastructure into a more coherent package hierarchy. The separate test.autoconfigure.properties package was collapsed into test.context.

How to Fix

A
Update the import. Replace org.springframework.boot.test.autoconfigure.properties.PropertyMapping with org.springframework.boot.test.context.PropertyMapping. This typically appears only in custom meta-annotations for test slices.
✓ Verify: mvn test-compile — no package does not exist error for @PropertyMapping import
Scope check: Search for @PropertyMapping and the old import path in your test source tree. Only affects teams that have written custom test slice annotations — most applications won't have this at all.

Watch Out

If you have a shared test-utilities library that defines custom slices, the fix must be applied there rather than in the consuming projects. A single unfixed library breaks all consumers.

Further Info
spring-break module: propertymapping-relocated· Spring Boot 4.0 Migration Guide
@PropertyMapping is used when authoring custom test slice annotations. It binds annotation attributes to Spring Boot auto-configuration properties. The move is part of the same test-infrastructure cleanup that also relocated several other test-autoconfigure classes in Spring Boot 4.0.
